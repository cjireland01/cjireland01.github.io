Software Design & Engineering Narrative

Original Artifact

The artifact I selected for enhancement is the Inventory Management Application I originally developed in October 2024. The application was created to streamline inventory tracking for a variety of use cases. While functional, the initial version relied on a local SQLite database with basic CRUD operations and lacked real-time capabilities or cloud connectivity. Because of this limitation, it was not suitable for real-world or commercial deployment, especially in scenarios requiring multiple devices or users accessing shared inventory data.

This original implementation served as a solid foundation, but its reliance on local-only data made it a strong candidate for enhancement. Given that I built the entire system independently and with minimal instructor guidance, I believed it was the best example to showcase my growth and technical abilities.

View the original SQLite database and the enhanced version with Firestore.

Enhancements Made

The most significant enhancement I made was migrating the application’s storage system from a local SQLite database to a cloud-based Firestore database through Firebase. This transformation fundamentally improved the application's scalability, enabling multiple users and devices to interact with a shared dataset in real time. The enhancement involved replacing the original DatabaseHelper.java class with a new cloud-integrated repository class called FirestoreInventoryRepository.java.

This new class streamlined inventory management through centralized, cloud-based data handling. One of the standout features is the listenToItems() method, which leverages Firestore’s addSnapshotListener to sync live updates to inventory data. Unlike the previous SQLite-based approach—which would have required custom polling mechanisms or broadcast receivers—Firestore provides real-time data synchronization out of the box.

Firestore’s additional benefits also strengthened the application’s architecture. Its built-in security rules eliminated the need to write extensive authentication and access control logic. Additionally, Firestore’s offline caching allows users to interact with data even without an active internet connection, making the application more robust and user-friendly.
