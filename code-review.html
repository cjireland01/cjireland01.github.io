Existing Functionality

Prior to the code adjustment, this application only worked for a single device. Multiple accounts were possible, but networking was not. This severely limited its use in commercial settings. This is primarily due to the use of SQLite as opposed to a cloud-based database.

This also struggled due to poorly optimized querying systems with overly complicated code.

Planned Enhancements

The code was changed to utilize Google's Firestore Firebase and authentication services with a focus on multiple locations. Sharding of the database and usage of Firestore's libraries enabled the source code to be much cleaner and more maintainable.

This change also allowed for administrative users, 3rd party authentication and account recovery, hundreds of networked devices and accounts, per-user inventory warning notifications, etc.
